# Agent naming convention

Lore agents use **thematic names** that evoke a role or character rather than plain functional
descriptions. `lore-librarian` and `session-finalizer` are the canonical examples: the name
signals disposition and intent without reading like a job-title abbreviation. Avoid names like
`note-writer` or `vault-updater` — those describe the mechanism, not the agent's character.
This convention keeps agent identifiers memorable and distinguishable when multiple agents are
active in the same session.
